## Architecture

Two script blocks, one HTML file:

| Block | Scope | Contains |
|---|---|---|
| Script 0 | `DOMContentLoaded` closure | `calculate()`, GPS, wind, carousel, club picker, course bar |
| Script 1 | `prepare-module` `'use strict'` | My Golf tab, stats, rounds, courses, score entry |

**Cross-scope rule:** Functions needed by both blocks must be on `window.*`. Use explicit `window.x = y` (not `Object.defineProperty` — Safari bug).

---

## Non-negotiable rules

### Calculation integrity

- **`_holeHcpAdj` must be set BEFORE `ordered[]` is built** in `calculate()` — moving it after breaks first-load HCP adjustment
- **Always pass `driverCarry` (not `teeClub.carry`)** to `expectedStrokesRemaining` — using teeClub carry causes tier mismatch between plans and overrides
- **`blendedScore` must be applied** in three places: `ordered[]` loop, `buildPlanWithShot2Override`, and `buildDeltaSection` / `getActivePlan`

### Override namespacing

All overrides keyed by `_hk(type)` = `courseId|holeIdx|strategyType`. Never use bare strategy type strings as keys.

```js
function _hk(type) { return _overrideCourseId + '|' + _overrideHoleIdx + '|' + type; }
```

`_overrideHoleIdx` and `_overrideCourseId` initialised from sessionStorage at parse time AND re-synced at start of `calculate()`.

### Wind / temperature

- `tempCarryFactor()` reads `windState.tempC` — actual air temp, NOT `feelsLike`
- Temperature correction only applies when wind is **enabled** — disabling wind disables temp too
- `applyWind()` returns raw carry immediately if `!windState.enabled`

### HTML structure

The wind panel nests 3 levels deep — easy to lose a closing tag:
```
windSection > collapsible-header + windLockHint + weatherCondRow + collapsible-body > windManualEntry + compassSection > windStatGrid
```
Missing `</div>` on `collapsible-body` will hide everything after it (including the entire My Golf tab).

## Storage keys

```
localStorage:
  golfBag_v2               — club bag setup
  golfCourses_v1           — all courses
  golfRounds_v1            — all rounds
  golfProfile_v1           — player name + handicap index
  golfWind_v1              — persisted hole direction
  golfTee_v1               — GPS tee mark
  roundScores_<courseId>   — in-progress round
  committedStrategies_<courseId>
  windEnabled

sessionStorage:
  activeCourse             — { id: string, holeIdx: number }
```

## Course IDs

5-digit numeric strings (10000–99999). Generated by `newCourseId()`, collision-checked. Older courses may have `c_timestamp_random` IDs — both work.

| `calculate(clearOverrides?)` | Script 0, DOMContentLoaded | Main entry point — wrapped by prepare-module |

## What NOT to do

- Don't use `teeClub.carry` as `driverCarry` parameter anywhere
- Don't set `_holeHcpAdj` after `ordered[]` is built
- Don't apply `blendedScore` to only some code paths — must be consistent
- Don't add `window` event listeners without cleanup (club picker drag had this bug)
- Don't use `passive: true` on `touchmove` where `preventDefault()` is needed
- Don't store overrides with bare strategy type as key — always use `_hk()`
- Don't display `feelsLike` as the temperature for carry calculations
