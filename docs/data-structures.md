# Data Structures

## localStorage keys

| Key | Type | Contents |
|---|---|---|
| `golfBag_v2` | object | Club selection, carry distances, conditions |
| `golfCourses_v1` | object map | All courses, keyed by course ID |
| `golfRounds_v1` | object map | All rounds, keyed by course ID → array |
| `golfProfile_v1` | object | Player name + handicap index |
| `golfWind_v1` | object | Persisted hole direction (`{ holeDeg }`) |
| `golfTee_v1` | object | GPS tee mark position |
| `roundScores_<courseId>` | array | In-progress round, 18 entries |
| `committedStrategies_<courseId>` | object | Per-hole committed strategy during active round |
| `windEnabled` | string `'true'|'false'` | Wind toggle state |

## sessionStorage keys

| Key | Contents |
|---|---|
| `activeCourse` | `{ id: string, holeIdx: number }` |

## Course object

Storage: `golfCourses_v1[courseId]`

```js
{
  name: string,
  courseRating: number,      // e.g. 72.1 — required for WHS model
  slopeRating: number,       // e.g. 125 — required for WHS model
  holes: Array(18) of {
    par: 3 | 4 | 5,
    si: number,              // stroke index 1–18 — all 18 required for WHS model
    length: number,          // metres
    note: string             // optional hole note
  }
}
```

### Course IDs
5-digit numeric strings (`10000`–`99999`). Generated by `newCourseId()`, collision-checked. Legacy courses may have `c_<timestamp>_<random>` IDs — both formats are supported throughout.

---

## Round object

Storage: `golfRounds_v1[courseId][]` — newest first (unshift on save)

```js
{
  date: string,              // 'YYYY-MM-DD'
  holesPlayed: number,       // 1–18
  totalStrokes: number,
  totalPar: number,
  totalPutts: number,
  totalGIR: number,          // greens in regulation count
  scores: Array(18) of null | {
    fairway: number,         // fairway shots taken
    rough: number,           // 0 or 1 — flag, not added to stroke count
    putts: number,
    gir: boolean,
    fir: boolean | null      // null on par 3
  },
  strategies: {
    [holeIdx: number]: string  // encoded: 'Max distance · Driver'
  }
}
```

### Score arithmetic
```
totalStrokes = fairway + rough + putts   // rough IS counted in total
```
Note: `rough` adds to total strokes (it's a shot), but it also separately triggers the +0.35 rough penalty in expected strokes when `window._inRough` is true during live play.

### GIR formula
```
gir = (fairway + rough) === (par - 2)
```

### FIR formula
```
if par <= 3:  fir = null       // not applicable
if rough > 0: fir = false      // any rough shot = missed fairway
if fairway > 0: fir = true
else: fir = null               // not yet entered
```

## In-progress scores

Storage: `roundScores_<courseId>` — array of 18 entries, each `null` or score object as above.

Persisted to localStorage (survives iOS app kill/background). Cleared when round is saved or cancelled.

## Bag object

Storage: `golfBag_v2`

```js
{
  checked: boolean[],        // 20 entries matching clubs array order
  driver: string,            // carry distance in metres
  i7: string,                // 7i carry distance
  pw: string,                // PW carry distance
  conditions: 'soft' | 'firm'
}
```

## Committed strategies

Storage: `committedStrategies_<courseId>` — JSON object

```js
{
  [holeIdx: number]: string  // encoded strategy, e.g. 'Max distance · Driver'
}
```

Decoded by `decodeStrategy(stored)`:
```js
// 'Max distance · Driver' → { type: 'Max distance', club: 'Driver' }
// 'Conservative'          → { type: 'Conservative', club: null }
```

---

## Strategy encoding

Strategy strings in `round.strategies` and `committedStrategies` use the format:
```
'<type>'               // e.g. 'Conservative'
'<type> · <club>'      // e.g. 'Max distance · Driver'
'Par 3 · <club>'       // e.g. 'Par 3 · 7i'
```

Strategy type values: `'Max distance'`, `'Controlled'`, `'Conservative'`. Legacy values `'Long'`, `'Medium'`, `'Short'`, `'Aggressive'`, `'Balanced'`, `'Safe'` may appear in older rounds and are handled in display code.
